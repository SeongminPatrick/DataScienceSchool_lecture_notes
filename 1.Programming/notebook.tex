\section{Thread}\label{thread}}

    프로그램을 실행하면서 시간이 걸린다면 대부분 아래 두가지중 한가지의
이유이다 - I/O 바운드 : 대부분의 경우 (data Input/Output) - CPU 바운드 :
계산량이 굉장히 많을 경우

    thread를 통해 성능을 향상시킬 수 있다

    \textbf{Synchronous 동기} - 한가지가 끝나야 다음 작업 수행 - 순서대로 -
데이터가 오고 가는 idle time을 비워놓고 기다린다

\textbf{Asynchronous 비동기} - 결과는 상관 없이 독립적으로 수행 후, -
결과를 순서대로 리턴하지는 않는다. - 응답이 오는 대로 저장 - 병렬로 동시
처리 가능 - 1번의 idle time 때 2번 처리를 요청한다 (기다리지 않는다) -
ex) 먼저 끝나는 대로 저장 - ex) 크롤링, 자동 업로드 때 사용 등등

결론 : idle time을 줄이자

싱글thread, 멀티thread : cpu 사용 \ldots{} 싱글은 타임라인 하나를
순서대로. 멀티thread는 타임라인을 쪼개서 idle을 활용
